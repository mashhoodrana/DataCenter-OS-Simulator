   ```bash

   ```
3. Create a **new branch** for your feature or fix

   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Pick an issue from the **Issues** tab (preferably labeled `good first issue` or `help wanted`)
5. Implement your solution following the project guidelines
6. **Commit** your changes with a meaningful message

   ```bash
   git commit -m "Add Round Robin scheduling module"
   ```
7. **Push** your branch to GitHub

   ```bash
   git push origin feature/your-feature-name
   ```
8. Open a **Pull Request (PR)** to the `main` branch
